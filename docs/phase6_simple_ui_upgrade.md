# Phase 6 Simple UI Upgrade

## 1. 目的

第6フェーズでは、第4フェーズのポップUIを土台にしつつ、毎日使う自治体ごみアプリとしての見やすさを優先し、シンプルで迷わないレイアウトへ再設計した。

## 2. シンプルUI方針

- 派手な装飾よりも、日付、地区、収集予定、分別方法を読みやすくする。
- 背景はごく薄いグレー、ヘッダーは落ち着いた緑、カードは白を基本にする。
- 色は意味補助に限定し、アイコンと日本語ラベルを併用する。
- カード過多を避け、直近予定や検索結果はリスト形式で表示する。
- 角丸、影、グラデーションを抑え、罫線と余白で情報を整理する。

## 3. 主な変更

- `AppColor` を落ち着いた緑・グレー中心に再調整。
- `AppTypography` の見出しを控えめにし、太さを読みやすい範囲に整理。
- `AppRadius` と `AppShadow` を弱め、自治体アプリらしい簡潔なカード表現に変更。
- ホームを「地区情報」「今日のごみ」「直近7日間」「よく使う機能」「注意」「データ」の流れに再構成。
- 検索を「大きな検索欄」「カテゴリ絞り込み」「空状態」「結果リスト」に整理。
- ごみ分別を主要タブ化し、カテゴリグリッドから詳細へ進む導線を明確化。
- カレンダーは月表示と選択日詳細を中心に、日付セルを大きめに調整。
- 設定は緑ヘッダーとリスト寄りカードで統一。

## 4. 維持した機能

- 門真市限定方針。
- A-F地区対応。
- 大倉町1-20=A地区プリセット。
- 分別カテゴリ、検索辞書、出し方データ。
- `GarbageSummaryProvider`。
- 通知予定プレビュー。
- master生成、schema validation、manifest/SHA。
- ライトモード専用設定。

## 5. コピーではないこと

添付画像から取り入れたのは、緑ヘッダー、直近予定リスト、検索空状態、カテゴリグリッド、シンプルな月カレンダーという情報設計の考え方である。画像内のアプリ名、キャラクター、固有アイコン、ブランド要素は使用していない。

## 6. 残課題

- Windows環境ではXcode PreviewとSimulatorの目視確認はできない。
- 実機でのVoiceOver、Dynamic Type最大、iPhone SE相当表示、通知許可/拒否フローは別途確認が必要。
