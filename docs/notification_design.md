# Notification Design

## 1. 目的

通知は便利機能ではなく、出し忘れ防止の信頼機能として扱う。誤通知を減らし、ユーザーが通知予定を事前に確認できる状態にする。

## 2. 通知対象

- 粗大ごみは要予約のため通常通知しない。
- カテゴリごとのON/OFF設定を尊重する。
- 前日夜通知と当日朝通知を分ける。
- 直近60日分を対象にする。

## 3. 通知予定プレビュー

設定画面に直近の通知予定を表示する。表示内容は以下。

- 通知タイミング
- 通知予定日時
- 通知タイトル
- 対象日
- ごみ種別アイコン

## 4. 再スケジュール

- 地区変更時に通知を再設定する。
- 大倉町1-20プリセット適用時に通知を再設定する。
- マスタ更新成功時に通知を再設定する。
- ユーザーが明示的に「通知を許可して再設定」を押せる。

## 5. 許可状態

通知許可状態は設定画面に表示する。拒否時はiOS設定を開く導線を出す。

## 6. 未検証項目

Windows環境では実機の通知許可ダイアログ、バックグラウンド通知、端末の集中モード、実通知到達は未検証。実機確認時に必ず確認する。
