# Developer Notification Testing

## 1. 目的

実機で通知許可、ローカル通知到達、通知文面、pending通知状態を確認しやすくする。ただし一般ユーザーの設定画面を複雑にしない。

## 2. 表示条件

- DEBUGビルドでは開発者用セクションを表示する。
- Release相当では、設定画面のバージョン番号を7回タップすると開発者用セクションを表示する。

## 3. 機能

- 通知許可状態の表示。
- 通知許可リクエスト。
- 5秒後のテスト通知。
- 10秒後のテスト通知。
- 明日のごみ通知の模擬。
- 今日の朝通知の模擬。
- pending通知一覧の表示。
- テスト通知だけのキャンセル。
- すべてのpending通知キャンセル。ただし確認ダイアログ付き。

## 4. 通常通知への影響防止

テスト通知identifierは `dev-test-notification-` で始める。テスト通知だけをキャンセルする操作はこのprefixだけを対象にする。通常通知は `gomi_` prefix のまま維持する。

## 5. 実機確認手順

1. 設定から開発者用画面を開く。
2. 通知許可状態を確認する。
3. 通知許可をリクエストする。
4. 5秒後テスト通知を押す。
5. ロック画面または通知センターで通知到達を確認する。
6. pending通知一覧を再読み込みする。
7. テスト通知だけ削除し、通常通知が残ることを確認する。

## 6. 未確認項目

Windows環境では実機通知到達、通知許可拒否からの復帰、バックグラウンド状態での通知到達は未確認。
