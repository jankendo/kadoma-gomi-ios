# Device Notification Test Plan

## 1. 目的

実機でローカル通知の許可、拒否、再設定、通知文面、通知予定プレビューを確認するための手順です。

## 2. 事前準備

- 最新の unsigned IPA または Xcode から実機へインストールする。
- 端末の日時とタイムゾーンを日本時間にする。
- アプリを一度削除して通知許可状態をリセットできるようにする。

## 3. 通知許可の確認

1. アプリを初回起動する。
2. 設定タブを開く。
3. 「通知を許可して再設定」を押す。
4. iOS の通知許可ダイアログで許可する。
5. 設定画面に「通知は許可されています」が表示されることを確認する。

## 4. 通知拒否の確認

1. アプリを削除して再インストールする。
2. 通知許可ダイアログで拒否する。
3. 設定画面に iOS設定を開く導線が表示されることを確認する。
4. iOS設定で通知を許可し、アプリに戻って再設定できることを確認する。

## 5. 通知予定プレビュー

- 前日通知は「明日は...の収集予定です」。
- 当日朝通知は「今日の朝は...の収集予定です」。
- 粗大ごみは通常通知に出ない。
- 通知本文に「地区設定に基づく通知です。変更がある場合は公式情報も確認してください。」が含まれる。

## 6. 地区変更時

1. 大倉町1-20 / A地区を設定する。
2. 通知予定を確認する。
3. B-F の任意地区に変更する。
4. 「通知を再設定します」の案内が出ること、通知予定が新地区基準に変わることを確認する。

## 7. マスタ更新時

1. 設定からマスタ更新を確認する。
2. 更新成功または最新版のメッセージを確認する。
3. 更新後に通知再設定メッセージが出ることを確認する。

## 8. 年末年始・未確認日の扱い

現時点では2026年12月から2027年1月の確定停止・振替日は未確認です。未確認日を確定通知として扱わず、年末年始注意文と公式確認導線が見えることを確認します。

## 9. 合格基準

- 通知許可状態が正しく表示される。
- 拒否時にiOS設定へ進める。
- 通知予定プレビューの日時・カテゴリがカレンダーと一致する。
- 粗大ごみが通常通知に混ざらない。
- 地区変更後に通知予定が変わる。
- 文面が自然な日本語で、公式確認が必要な時期を誤解させない。
