# Calendar Simple Layout

## 1. 目的

カレンダーは、高機能に見せるより、日付と収集予定をすぐ読めることを優先する。

## 2. 構成

1. 月ヘッダー
2. 月/リスト表示切替
3. 公式確認注意
4. 凡例
5. 月表示グリッド
6. 選択日の詳細

## 3. 月ヘッダー

現在月、地区名、前月/次月、今日ボタンをまとめる。説明文は短くし、日付を選ぶと下に詳細が出ることを伝える。

## 4. 日付セル

- セルを大きめにする。
- 今日を緑の丸で表示する。
- 選択日は緑の枠線で表示する。
- 収集予定は小さなアイコンと短いラベルで表示する。
- 複数件は2件まで表示し、残数を `+n` で示す。

## 5. 凡例

凡例はアイコンと短い名前で表示する。色だけで意味を伝えない。

## 6. 選択日詳細

選択日の収集予定を下部カードに表示する。収集なしの日は「この日は収集予定なし」と明記する。

## 7. 年末年始・例外日

12月・1月は未確認変更の可能性を注意表示する。confirmedではない例外日を確定収集日として断定しない。

## 8. 小型画面対応

iPhone SE相当ではセル内の情報を絞り、詳細は選択日カードに逃がす。Dynamic Type大ではカレンダーチップの文字が縮小されるが、カテゴリ名は選択日詳細で読める。

## 9. Phase 7: 月間カレンダーのみ

第7フェーズで、選択日詳細、凡例カード、リスト表示、表示切替を削除した。カレンダータブは月間グリッドを画面の主役にする。

小型画面ではセル内の表示件数をさらに絞り、詳細説明は検索タブとごみ分別タブへ分離する。VoiceOverでは日付セルごとに日付と収集予定を読み上げる。
