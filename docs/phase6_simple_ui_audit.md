# Phase 6 Simple UI Audit

## 1. 現在のUI実装概要

第5フェーズ時点のアプリは、門真市限定、A-F地区対応、大倉町1-20=A地区プリセット、225品目以上の検索辞書、分別ガイド、通知予定プレビュー、master生成、schema validation、ライトモード固定を備えている。UIは第4フェーズのポップで明るい方向をベースに、ホーム、検索、分別ガイド、カレンダー、設定がSwiftUIで構成されている。

## 2. 参考画像から取り入れる設計思想

- 上部は落ち着いた緑のヘッダーにし、画面タイトルを明確にする。
- ホームは直近の日付ごとの収集予定を縦に並べ、毎朝すぐ読める構成にする。
- 検索は大きい検索欄と明確な空状態に絞る。
- 分別はカテゴリ入口をアイコン付きグリッドで整理する。
- カレンダーはセルを大きくし、日付と収集情報の関係を読みやすくする。

参考画像のアプリ名、固有キャラクター、独自アイコン、配色そのものは使用しない。

## 3. UI/UX上の残課題

- ポップなグラデーション、丸み、影が多く、生活インフラアプリとしては情報より装飾が目立つ箇所があった。
- ホームの今日/明日/次回情報は豊富だが、直近1週間を素直に確認する流れが弱かった。
- 検索画面にガイド要素が多く、初期状態で「入力すればよい」という行動がやや埋もれていた。
- 分別カテゴリは存在するが、主要タブの入口としては整理し直す余地があった。
- カレンダーは高機能だが、セル内の文字と装飾が多く、小型画面では読み疲れしやすい可能性があった。

## 4. デザインシステム上の課題

- `AppTypography` の `heavy` 指定が多く、全体が強く見えやすかった。
- `AppRadius` の角丸が大きめで、ポップ寄りの印象が強かった。
- `AppShadow` とグラデーションが複数箇所で残り、自治体アプリ風の簡潔さから離れていた。
- カードUIが多く、見出し、罫線、リストで整理する方が適している画面があった。

## 5. アクセシビリティ上のリスク

- 色とバッジが多い画面では、色弱ユーザーや高齢者が意味を取りにくい可能性がある。
- カレンダーセル内の短いチップはDynamic Type最大では詰まりやすい。
- タップ先が多いカードでは、VoiceOver読み上げ時に目的が伝わりにくい可能性がある。

## 6. 実装方針

- データ、検索、通知、master生成、CI/CDは維持し、UIレイヤー中心に改修する。
- タブは「ホーム / ごみ検索 / ごみ分別 / カレンダー / 設定」に整理する。
- `BulkyWasteView`は残すが、主要入口は分別ガイドのカテゴリに統合する。
- 色数を抑え、緑ヘッダー、白カード、薄い緑/グレー背景、控えめなアクセントに統一する。
- Dynamic TypeとVoiceOverを前提に、短い文言、大きい行高、明確なラベルを優先する。
