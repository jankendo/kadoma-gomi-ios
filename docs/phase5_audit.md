# Phase 5 監査

確認日: 2026-05-25

## 1. 現在の実装概要

門真市限定の SwiftUI iOS アプリ。A-F全地区、2026年度収集ルール、225品目の検索辞書、分割JSONからのmaster/manifest生成、ライトモード専用のポップUI、通知予定プレビューを保持している。

## 2. データモデルの状態

既存の `WasteCategory` はカテゴリ名、色、アイコン、通知既定値、短い出し方を保持していた。`WasteItem` は品目名、別名、検索キーワード、分別カテゴリ、注意フラグ、公式確認推奨フラグを保持していた。

Phase 5前は、検索結果で「何ごみか」は分かるが、「出す前に何をするか」を品目詳細として深く説明する構造が弱かった。

## 3. 検索UIの状態

225品目、表記ゆれ、危険物・粗大候補・公式確認推奨バッジに対応済み。検索結果カードは読みやすいが、詳細画面への遷移とステップ表示が不足していた。

## 4. ホームUIの状態

今日・明日・次回の収集予定は分かりやすい。Phase 5前は、今日出せるごみの「代表例」「出し方の要点」「注意点」がホーム上で十分には見えなかった。

## 5. カテゴリ表示の状態

カテゴリ絞り込みはあったが、カテゴリ単位の分別ガイド画面は不足していた。引っ越し直後や高齢者が「普通ごみとは何か」「古紙はどう出すか」を一覧から確認する導線が弱かった。

## 6. 品質リスク

- 公式確認済み情報と要確認情報の区別がUIに十分出ない可能性
- 品目数が増えたことで検索結果後の理解負荷が上がる可能性
- ホームに情報を増やすと小型画面で詰まる可能性
- 新規SwiftUIファイルがXcode projectに入っていないとCIビルドで落ちる可能性

## 7. 今回の実装方針

- 門真市公式の「ごみの出し方・分け方」と共通PDFを根拠に、カテゴリ別の出し方ステップを追加する
- 225品目は維持し、各品目にカテゴリ由来の手順とチェック項目を付与する
- 検索結果から詳細画面へ遷移できるようにする
- ホームで今日または次回のごみの代表例・出し方・注意点を表示する
- 画像は無断取得せず、SF Symbols中心で統一する
